A API do LF Connect aplica limites pra proteger a plataforma e os clientes finais.
Conheça os números antes de planejar integrações que façam muitas chamadas.

## Rate limit padrão

| Recurso | Limite |
| --- | --- |
| Requisições por token | 60 / minuto |
| Mensagens outbound via WhatsApp | 30 / minuto por inbox |
| Mensagens via WhatsApp Cloud API | Conforme tier Meta da conta |
| Upload de mídia (multipart) | 10 / minuto |
| Importação CSV (contatos / Kanban) | 1 job por vez por conta |

## Headers de resposta

Toda resposta retorna headers que te ajudam a controlar o ritmo:

```http
X-RateLimit-Limit: 60
X-RateLimit-Remaining: 47
X-RateLimit-Reset: 1715972400
```

- **`X-RateLimit-Limit`** — teto da janela atual
- **`X-RateLimit-Remaining`** — quantas ainda sobram
- **`X-RateLimit-Reset`** — timestamp Unix de quando o contador zera

## Quando você é bloqueado

Se estourar o limite, recebe **`429 Too Many Requests`**:

```json
{
  "error": "Rate limit exceeded. Try again in 23 seconds."
}
```

<Warning>
  Não tente "burlar" rate limit com múltiplos tokens do mesmo usuário —
  o sistema rastreia também por IP e por conta. Em vez disso, [fale com nossa
  equipe](mailto:contato@lfautomacoes.com) sobre seu caso de uso.
</Warning>

## Limites de mídia

Quando enviar anexos via API, respeite os tetos do WhatsApp:

| Tipo de mídia | Limite |
| --- | --- |
| Imagem | 5 MB |
| Áudio | 16 MB |
| Vídeo | 32 MB |
| Documento | 100 MB |
| Sticker | 500 KB |

## Boas práticas

<AccordionGroup>
  <Accordion title="Use webhooks em vez de polling">
    Não fique chamando `GET /conversations` a cada 5 segundos esperando uma mensagem nova.
    Configure um [webhook](/essentials/webhooks) e seja notificado quando acontecer.
  </Accordion>
  <Accordion title="Faça batch nas operações em massa">
    Pra mover 100 cards no Kanban, use o endpoint `bulk_move_items` em vez de 100 chamadas individuais.
  </Accordion>
  <Accordion title="Cacheie identificadores estáticos">
    `account_id`, `inbox_id`, `funnel_id`, `team_id` quase nunca mudam. Carregue 1 vez no startup do seu serviço e reuse.
  </Accordion>
  <Accordion title="Respeite o `Retry-After`">
    Quando receber 429, o header `Retry-After` te diz exatamente em segundos quando voltar a tentar. Use isso, não chute.
  </Accordion>
</AccordionGroup>
