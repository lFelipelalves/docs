---
title: "Campanhas"
description: "Disparos automáticos (ongoing) e em massa (one-off) via API."
---

Campanhas permitem disparar mensagens em massa pra grupos de contatos:

- **Ongoing** — dispara automaticamente quando um visitante atinge regras configuradas (ex: passou X tempo no site)
- **One-off** — disparo manual pra audiência definida na hora

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---


## Criar campanha **ongoing** (widget)

```http
POST /campaigns
```

<ParamField body="title" type="string" required>Nome interno da campanha</ParamField>
<ParamField body="message" type="string" required>Texto enviado</ParamField>
<ParamField body="inbox_id" type="integer" required>ID da inbox Webchat</ParamField>
<ParamField body="sender_id" type="integer" required>Agente que aparece como remetente</ParamField>
<ParamField body="campaign_type" type="enum" default="ongoing">`ongoing` ou `one_off`</ParamField>
<ParamField body="trigger_only_during_business_hours" type="boolean">Só dispara em horário comercial</ParamField>
<ParamField body="trigger_rules" type="object">Regras de disparo (ver abaixo)</ParamField>

#### `trigger_rules` (ongoing)

<ParamField body="trigger_rules.url" type="string">URL exata onde dispara</ParamField>
<ParamField body="trigger_rules.time_on_page" type="integer">Segundos no site antes de disparar</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/campaigns \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Saudação na página de preços",
    "message": "Oi! Está com dúvida sobre os planos? Posso ajudar.",
    "inbox_id": 2,
    "sender_id": 7,
    "campaign_type": "ongoing",
    "trigger_only_during_business_hours": true,
    "trigger_rules": {
      "url": "https://lfautomatiza.com/precos",
      "time_on_page": 30
    }
  }'
```

### Resposta `201`

```json
{
  "id": 14,
  "title": "Saudação na página de preços",
  "campaign_type": "ongoing",
  "campaign_status": "draft",
  "inbox_id": 2,
  "sender_id": 7,
  "created_at": "2026-05-18T22:30:00Z"
}
```

## Criar campanha **one-off** (envio em massa)

```http
POST /campaigns
```

<ParamField body="title" type="string" required>Nome interno</ParamField>
<ParamField body="message" type="string" required>Texto com variáveis (`{{contact.name}}`...)</ParamField>
<ParamField body="inbox_id" type="integer" required>Inbox de envio</ParamField>
<ParamField body="campaign_type" type="enum" required>`one_off`</ParamField>
<ParamField body="audience" type="array" required>Lista de objetos `{ type, id }`</ParamField>
<ParamField body="scheduled_at" type="datetime">ISO 8601 — quando disparar (omita pra disparar na hora)</ParamField>

#### Variáveis disponíveis em `message`

| Variável | Valor |
| --- | --- |
| `{{contact.name}}` | Nome do contato |
| `{{contact.email}}` | E-mail |
| `{{contact.phone_number}}` | Telefone |
| `{{contact.identifier}}` | ID externo |
| `{{contact.custom_attributes.X}}` | Valor de atributo personalizado |

#### Spintax (variações automáticas)

```
{Olá|Oi|E aí} {{contact.name}}! Temos {uma novidade|uma oferta} pra você.
```

A plataforma escolhe aleatoriamente uma variação em cada envio.

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/campaigns \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Promoção Mai/2026",
    "message": "Oi {{contact.name}}, temos uma novidade pra você!",
    "inbox_id": 1,
    "campaign_type": "one_off",
    "audience": [
      { "type": "Label", "id": 1 },
      { "type": "Label", "id": 2 }
    ],
    "scheduled_at": "2026-05-20T10:00:00-03:00"
  }'
```

### Resposta `201`

```json
{
  "id": 15,
  "title": "Promoção Mai/2026",
  "campaign_type": "one_off",
  "campaign_status": "scheduled",
  "scheduled_at": "2026-05-20T10:00:00-03:00",
  "audience_total": 1247
}
```

## Atualizar campanha

```http
PATCH /campaigns/{id}
```

<ParamField path="id" type="integer" required>ID da campanha</ParamField>

Só funciona enquanto a campanha está em `draft`, `paused` ou `scheduled`. Campanhas
em execução não podem ser editadas.

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/campaigns/15 \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Oi {{contact.name}}, novidade exclusiva pra você!",
    "scheduled_at": "2026-05-21T10:00:00-03:00"
  }'
```

### Resposta `200`

```json
{
  "id": 15,
  "message": "Oi {{contact.name}}, novidade exclusiva pra você!",
  "scheduled_at": "2026-05-21T10:00:00-03:00",
  "updated_at": "2026-05-18T22:45:00Z"
}
```

## Pausar campanha

```http
POST /campaigns/{id}/pause
```

<ParamField path="id" type="integer" required>ID da campanha</ParamField>

Útil pra parar uma campanha já em execução se detectar problema (alta taxa de
falha, bloqueio, etc.).

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/campaigns/15/pause \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "id": 15,
  "campaign_status": "paused",
  "paused_at": "2026-05-20T11:00:00Z",
  "stats_at_pause": { "sent": 423, "failed": 12, "remaining": 812 }
}
```

## Excluir campanha

```http
DELETE /campaigns/{id}
```

<ParamField path="id" type="integer" required>ID da campanha</ParamField>

Só permitido em `draft`. Campanhas executadas viram histórico (preservadas).

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/campaigns/15 \
  -H "api_access_token: $TOKEN"
```

### Resposta `204`

Sem corpo.

## Status da campanha (`campaign_status`)

| Status | Significado |
| --- | --- |
| `draft` | Criada mas não disparada |
| `active` | Ongoing — disparando conforme triggers |
| `scheduled` | One-off aguardando `scheduled_at` |
| `running` | One-off em execução |
| `completed` | One-off finalizada |
| `paused` | Pausada manualmente |
| `failed` | Falhou globalmente |

---

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem / leitura / atualização / pause / resume |
| `201 Created` | Campanha criada |
| `204 No Content` | Excluída |
| `403 Forbidden` | Apenas administrador |
| `422 Unprocessable Entity` | Inbox inválida, audiência vazia, template não aprovado |
