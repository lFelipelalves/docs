---
title: "Equipes e agentes"
description: "Gestão de times e atendentes da conta."
---

**Agentes** são os atendentes individuais da conta. **Equipes** (teams) agrupam
agentes por especialidade ou departamento — útil pra rotear conversas por área
(Vendas, Suporte, Financeiro).

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

# Agentes

## Listar agentes

```http
GET /agents
```

### Requisição

```bash cURL
curl https://chat.lfautomatiza.com/api/v1/accounts/1/agents \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
[
  {
    "id": 7,
    "name": "Sabrina",
    "email": "sabrina@lfautomatiza.com",
    "role": "administrator",
    "availability_status": "online",
    "confirmed": true,
    "auto_offline": true,
    "thumbnail": "https://cdn.lfautomatiza.com/users/7/avatar.png",
    "created_at": "2026-04-01T10:00:00Z"
  },
  {
    "id": 11,
    "name": "Marcos",
    "email": "marcos@lfautomatiza.com",
    "role": "agent",
    "availability_status": "busy",
    "confirmed": true
  }
]
```

---

## Convidar agente

```http
POST /agents
```

<ParamField body="email" type="string" required>E-mail do novo agente</ParamField>
<ParamField body="name" type="string">Nome</ParamField>
<ParamField body="role" type="enum" required>`administrator` ou `agent`</ParamField>

O agente recebe e-mail com link de ativação. Até ativar, fica como `confirmed: false`.

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/agents \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "novo.atendente@empresa.com",
    "name": "Maria Souza",
    "role": "agent"
  }'
```

### Resposta `201`

```json
{
  "id": 22,
  "email": "novo.atendente@empresa.com",
  "name": "Maria Souza",
  "role": "agent",
  "confirmed": false,
  "invite_sent_at": "2026-05-18T21:00:00Z"
}
```

---

## Atualizar agente

```http
PATCH /agents/{id}
```

<ParamField path="id" type="integer" required>ID do agente</ParamField>
<ParamField body="role" type="enum">`administrator` ou `agent`</ParamField>
<ParamField body="availability" type="enum">`online`, `busy`, `offline`</ParamField>

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/agents/7 \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "role": "administrator" }'
```

### Resposta `200`

```json
{
  "id": 7,
  "role": "administrator",
  "updated_at": "2026-05-18T21:15:00Z"
}
```

---

## Remover agente

```http
DELETE /agents/{id}
```

<ParamField path="id" type="integer" required>ID do agente</ParamField>

Remove o agente da conta. Conversas atribuídas a ele ficam sem agente (precisam ser
reatribuídas manualmente ou via regra).

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/agents/22 \
  -H "api_access_token: $TOKEN"
```

### Resposta `204`

Sem corpo.

## Disponibilidade do agente (`availability_status`)

| Status | Significado |
| --- | --- |
| `online` | Disponível pra receber conversas (auto-assignment elegível) |
| `busy` | Ocupado (não recebe novas atribuições automáticas) |
| `offline` | Desconectado |

## Papéis (`role`)

| Role | Permissões |
| --- | --- |
| `administrator` | Tudo: configurações, gestão de equipe, inboxes, integrações |
| `agent` | Só atende — não muda configurações da conta |

Para granularidade maior, planos enterprise permitem **funções personalizadas** (custom roles).

---

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Sucesso em GET / PATCH |
| `201 Created` | Agente convidado / equipe criada |
| `204 No Content` | Removido com sucesso |
| `403 Forbidden` | Apenas administrador pode gerenciar agentes/equipes |
| `404 Not Found` | Recurso não existe |
| `422 Unprocessable Entity` | E-mail já vinculado a outra conta |
