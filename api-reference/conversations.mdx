title: "Conversas"
description: "CRUD de conversas, atribuição, labels, status, prioridade e filtros avançados."

Conversas (também chamadas de tickets) são o núcleo da API do LF Connect.
Cada conversa pertence a uma **inbox** (canal) e é vinculada a um **contato**.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Listar conversas

```http
GET /conversations
```

<ParamField query="status" type="enum">`open`, `resolved`, `pending`, `snoozed`, `all`</ParamField>
<ParamField query="assignee_type" type="enum">`me`, `unassigned`, `assigned`, `unassigned_or_me`</ParamField>
<ParamField query="inbox_id" type="integer">Filtra por inbox</ParamField>
<ParamField query="team_id" type="integer">Filtra por equipe</ParamField>
<ParamField query="labels[]" type="array">Uma ou mais labels (combinação AND)</ParamField>
<ParamField query="q" type="string">Busca textual</ParamField>
<ParamField query="page" type="integer" default="1">Paginação (25 itens por página)</ParamField>

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v1/accounts/1/conversations?status=open&assignee_type=me" \
```

## Criar conversa

```http
POST /conversations
```

<ParamField body="source_id" type="string" required>Identificador externo (ex: número WhatsApp)</ParamField>
<ParamField body="inbox_id" type="integer" required>Inbox onde a conversa nasce</ParamField>
<ParamField body="contact_id" type="integer" required>Contato vinculado</ParamField>
<ParamField body="status" type="enum" default="open">`open` ou `pending`</ParamField>
<ParamField body="assignee_id" type="integer">Agente responsável inicial</ParamField>
<ParamField body="team_id" type="integer">Equipe responsável</ParamField>
<ParamField body="additional_attributes" type="object">Metadata sistêmica</ParamField>
<ParamField body="custom_attributes" type="object">Atributos personalizados</ParamField>
<ParamField body="message" type="object">Primeira mensagem (opcional, ver objeto abaixo)</ParamField>

#### `message`

<ParamField body="message.content" type="string" required>Texto</ParamField>
<ParamField body="message.message_type" type="enum">`outgoing` (padrão) ou `incoming`</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/conversations \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "source_id": "5511999999999",
    "inbox_id": 1,
    "contact_id": 542,
    "status": "open",
    "assignee_id": 7,
    "message": {
      "content": "Olá! Em que posso ajudar?",
      "message_type": "outgoing"
    }
  }'
```

### Resposta `200`

```json
{
  "id": 1290,
  "display_id": 125,
  "status": "open",
  "inbox_id": 1,
  "contact_id": 542,
  "assignee_id": 7,
  "created_at": 1715990000
}
```

## Mudar status

```http
POST /conversations/{id}/toggle_status
```

<ParamField path="id" type="integer" required>ID da conversa</ParamField>
<ParamField body="status" type="enum">`open`, `resolved`, `pending`, `snoozed` (se omitido, alterna entre open ↔ resolved)</ParamField>
<ParamField body="snoozed_until" type="datetime">Quando `status=snoozed`, define data de retorno (ISO 8601)</ParamField>

### Requisição — resolver

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/conversations/1287/toggle_status \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "status": "resolved" }'
```

### Requisição — snooze até data específica

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/conversations/1287/toggle_status \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "snoozed",
    "snoozed_until": "2026-05-25T14:00:00Z"
  }'
```

### Resposta `200`

```json
{
  "meta": { "all_count": 64 },
  "payload": {
    "id": 1287,
    "status": "resolved",
    "resolved_at": 1715990500
  }
}
```

## Filtro avançado

```http
POST /conversations/filter
```

Quando os query params do GET não chegam, use este endpoint com payload estruturado.

<ParamField body="payload" type="array" required>Array de condições (ver formato abaixo)</ParamField>

#### Estrutura de cada condição

<ParamField body="payload[].attribute_key" type="string" required>Atributo a consultar (ver lista abaixo)</ParamField>
<ParamField body="payload[].filter_operator" type="enum" required>Operador de comparação</ParamField>
<ParamField body="payload[].values" type="array" required>Valor(es) a comparar</ParamField>
<ParamField body="payload[].query_operator" type="enum">`and` ou `or` — como conectar com a próxima condição</ParamField>

#### Atributos consultáveis

`status`, `assignee_id`, `team_id`, `inbox_id`, `labels`, `priority`, `created_at`,
`last_activity_at`, `browser_language`, `country_code`, `referer`, `campaign_id`,
qualquer `custom_attributes.{chave}`.

#### Operadores

| Operador | Aplica-se a |
| --- | --- |
| `equal_to`, `not_equal_to` | Tudo |
| `contains`, `does_not_contain` | Strings, arrays |
| `greater_than`, `less_than` | Números, datas |
| `is_present`, `is_not_present` | Tudo (verifica null) |
| `is_greater_than`, `is_less_than` | Datas |

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/conversations/filter \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "payload": [
      {
        "attribute_key": "status",
        "filter_operator": "equal_to",
        "values": ["open"],
        "query_operator": "and"
      },
      {
        "attribute_key": "labels",
        "filter_operator": "contains",
        "values": ["vip"],
        "query_operator": "and"
      },
      {
        "attribute_key": "created_at",
        "filter_operator": "greater_than",
        "values": ["2026-05-01"]
      }
    ]
  }'
```

### Resposta `200`

```json
{
  "meta": { "count": 18 },
  "payload": [
    { "id": 1287, "status": "open", "labels": ["vip"], "messages_count": 8 }
  ]
}
```

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem, leitura, atualização |
| `201 Created` | Conversa criada |
| `204 No Content` | Atribuição / labels atualizadas |
| `400 Bad Request` | Payload inválido |
| `401 Unauthorized` | Token ausente/inválido |
| `403 Forbidden` | Usuário não tem acesso à conversa |
| `404 Not Found` | Conversa não existe |
| `422 Unprocessable Entity` | Inbox/contato inválido na criação |
