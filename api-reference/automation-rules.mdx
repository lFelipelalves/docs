---
title: "Regras de automação"
description: "CRUD de regras event-driven que automatizam ações na plataforma."
---

Regras de automação seguem o padrão **Evento → Condições → Ações**. Toda vez que
um evento configurado acontece, a plataforma avalia as condições — se passarem,
executa as ações.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Listar regras

```http
GET /automation_rules
```

### Requisição

```bash cURL
curl https://chat.lfautomatiza.com/api/v1/accounts/1/automation_rules \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "payload": [
    {
      "id": 21,
      "name": "Mensagem com boleto vai pro Financeiro",
      "description": "Roteia automaticamente menções a boleto",
      "event_name": "message_created",
      "active": true,
      "conditions": [
        {
          "attribute_key": "content",
          "filter_operator": "contains",
          "values": ["boleto"]
        }
      ],
      "actions": [
        { "action_name": "assign_team", "action_params": [3] },
        { "action_name": "add_label", "action_params": ["financeiro"] }
      ],
      "executions_count": 47,
      "created_at": "2026-04-12T10:00:00Z"
    }
  ]
}
```

## Criar regra

```http
POST /automation_rules
```

<ParamField body="name" type="string" required>Nome legível</ParamField>
<ParamField body="description" type="string">Contexto / motivo (recomendado em contas com muitas regras)</ParamField>
<ParamField body="event_name" type="enum" required>Quando dispara (ver tabela)</ParamField>
<ParamField body="conditions" type="array" required>Filtros (combinam com `query_operator`)</ParamField>
<ParamField body="actions" type="array" required>Ações executadas se passar</ParamField>
<ParamField body="active" type="boolean" default="true">Liga/desliga</ParamField>

### Eventos disponíveis (`event_name`)

| Evento | Quando dispara |
| --- | --- |
| `conversation_created` | Nova conversa aberta |
| `conversation_updated` | Qualquer campo da conversa muda |
| `message_created` | Mensagem nova (incoming ou outgoing) |
| `conversation_opened` | Status muda pra `open` (reabertura também) |

### Atributos consultáveis em `conditions`

`content`, `status`, `inbox_id`, `team_id`, `assignee_id`, `labels`, `priority`,
`message_type`, `country_code`, `browser_language`, `referer`, `campaign_id`,
qualquer `custom_attributes.{chave}`.

### Operadores em `conditions`

| Operador | Aplica-se a |
| --- | --- |
| `equal_to`, `not_equal_to` | Tudo |
| `contains`, `does_not_contain` | Strings, arrays |
| `greater_than`, `less_than` | Números |
| `is_present`, `is_not_present` | Tudo |
| `starts_with`, `ends_with` | Strings |

### Ações disponíveis (`action_name`)

| Ação | `action_params` |
| --- | --- |
| `assign_agent` | `[agent_id]` |
| `assign_team` | `[team_id]` |
| `assign_best_agent` | `[]` (round-robin entre disponíveis) |
| `remove_assigned_team` | `[]` |
| `add_label` | `["label1", "label2", ...]` |
| `remove_label` | `["label1", ...]` |
| `send_message` | `["Texto da resposta"]` |
| `send_email_to_team` | `[{ "team_ids": [3], "subject": "..." }]` |
| `change_status` | `["resolved"]` ou `["open"]`, `["pending"]`, `["snoozed"]` |
| `change_priority` | `["urgent"]` |
| `send_webhook_event` | `["https://meu-webhook.com/endpoint"]` |
| `mute_conversation` | `[]` |
| `snooze_conversation` | `["1_week"]` ou outra duração |
| `send_attachment` | `[attachment_id]` |
| `add_private_note` | `["Texto da nota interna"]` |

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/automation_rules \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mensagem com boleto vai pro Financeiro",
    "description": "Roteia automaticamente menções a boleto",
    "event_name": "message_created",
    "conditions": [
      {
        "attribute_key": "content",
        "filter_operator": "contains",
        "values": ["boleto"],
        "query_operator": "and"
      },
      {
        "attribute_key": "message_type",
        "filter_operator": "equal_to",
        "values": ["incoming"]
      }
    ],
    "actions": [
      { "action_name": "assign_team", "action_params": [3] },
      { "action_name": "add_label", "action_params": ["financeiro", "aguardando-pagamento"] },
      { "action_name": "change_priority", "action_params": ["high"] }
    ],
    "active": true
  }'
```

### Resposta `201`

```json
{
  "id": 22,
  "name": "Mensagem com boleto vai pro Financeiro",
  "event_name": "message_created",
  "active": true,
  "created_at": "2026-05-18T23:00:00Z"
}
```

## Clonar regra

```http
POST /automation_rules/{id}/clone
```

Útil pra testar variações sem mexer na original.

<ParamField path="id" type="integer" required>ID da regra a clonar</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/automation_rules/21/clone \
  -H "api_access_token: $TOKEN"
```

### Resposta `201`

```json
{
  "id": 23,
  "name": "Copy of: Mensagem com boleto vai pro Financeiro",
  "event_name": "message_created",
  "active": false,
  "cloned_from_id": 21,
  "created_at": "2026-05-18T23:30:00Z"
}
```

A regra clonada nasce com `active: false` por padrão — você revisa e ativa depois.

## Ordem de execução

Quando múltiplas regras casam com o mesmo evento, elas executam **na ordem em que
foram criadas** (mais antiga primeiro). Cada ação afeta o estado pras próximas
regras avaliarem.

<Tip>
  Use o campo `description` pra deixar claro o propósito de cada regra. Em contas
  com 20+ regras, o nome sozinho não conta a história toda.
</Tip>

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem / leitura / atualização / toggle |
| `201 Created` | Regra criada ou clonada |
| `204 No Content` | Excluída |
| `403 Forbidden` | Apenas administrador |
| `422 Unprocessable Entity` | Condição ou ação inválida |
