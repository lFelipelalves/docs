A API de relatórios retorna métricas agregadas em séries temporais ou totais.
Endpoints ficam no **prefixo `/api/v2/`** (não `/v1/` como o resto).

**Base:** `https://chat.lfautomatiza.com/api/v2/accounts/{account_id}/reports`

---

## Sumário geral

```http
GET /summary
```

<ParamField query="since" type="integer">Timestamp Unix de início</ParamField>
<ParamField query="until" type="integer">Timestamp Unix de fim</ParamField>
<ParamField query="type" type="enum">`account` (padrão), `agent`, `inbox`, `team`, `label`</ParamField>
<ParamField query="id" type="integer">ID do recurso quando `type ≠ account`</ParamField>

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v2/accounts/1/reports/summary?since=1715900000&until=1715972400" \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "open_conversations_count": 47,
  "agents_online_count": 5,
  "previous_period": {
    "open_conversations_count": 38,
    "agents_online_count": 4
  },
  "average_first_response_time": 142,
  "average_resolution_time": 1843,
  "average_response_time": 67,
  "conversations_count": 124,
  "incoming_messages_count": 387,
  "outgoing_messages_count": 412,
  "resolutions_count": 89
}
```

## Conversations report (série temporal)

```http
GET /conversations
```

<ParamField query="metric" type="enum" required>Métrica a calcular (ver tabela abaixo)</ParamField>
<ParamField query="since" type="integer" required>Timestamp Unix de início</ParamField>
<ParamField query="until" type="integer" required>Timestamp Unix de fim</ParamField>
<ParamField query="type" type="enum">`account`, `agent`, `inbox`, `team`, `label`</ParamField>
<ParamField query="id" type="integer">Quando `type ≠ account`</ParamField>
<ParamField query="group_by" type="enum">`day` (padrão), `week`, `month`, `year`</ParamField>
<ParamField query="business_hours" type="boolean">`true` calcula tempos apenas dentro do horário comercial</ParamField>

### Métricas disponíveis (`metric`)

- `conversations_count`
- `incoming_messages_count`
- `outgoing_messages_count`
- `avg_first_response_time`
- `avg_resolution_time`
- `resolutions_count`
- `bot_resolutions_count`
- `bot_handoffs_count`

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v2/accounts/1/reports/conversations?metric=conversations_count&since=1715900000&until=1715972400&type=account&group_by=day" \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
[
  { "value": 14, "timestamp": 1715817600 },
  { "value": 22, "timestamp": 1715904000 },
  { "value": 31, "timestamp": 1715990400 }
]
```

## Exportar como CSV

A maioria dos endpoints de sumário aceita query `?type=csv` pra retornar arquivo em
vez de JSON.

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v2/accounts/1/reports/conversations?metric=conversations_count&since=1715900000&until=1715972400&type=csv" \
  -H "api_access_token: $TOKEN" \
  -o relatorio.csv
```

### Resposta `200`

CSV em **UTF-8 com BOM** + separador `;`:

```csv
timestamp;value
1715817600;14
1715904000;22
1715990400;31
```

Abre direto no Excel BR sem quebrar acentuação.

---

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Métrica retornada |
| `400 Bad Request` | Janela de tempo inválida ou parâmetro desconhecido |
| `403 Forbidden` | Apenas administrador acessa relatórios |
| `422 Unprocessable Entity` | Combinação inválida (ex: `type=agent` sem `id`) |
