---
title: "Funis (Funnels)"
description: "CRUD de funis, métricas por etapa e reordenação."
---

Funis são os pipelines do seu Kanban. Você pode ter quantos quiser (Vendas SMB,
Vendas Enterprise, Renovação, Onboarding...), cada um com etapas próprias.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---


<ParamField path="account_id" type="integer" required>ID da conta no LF Connect</ParamField>

## Criar funil

```http
POST /funnels
```

### Corpo da requisição

<ParamField body="funnel.name" type="string" required>Nome do funil</ParamField>
<ParamField body="funnel.description" type="string">Texto livre descrevendo o pipeline</ParamField>
<ParamField body="funnel.active" type="boolean" default="true">`false` arquiva sem deletar</ParamField>
<ParamField body="funnel.is_default" type="boolean" default="false">Funil padrão ao abrir o Kanban</ParamField>
<ParamField body="funnel.stages" type="object" required>Mapa de etapas (chave = `id` do stage)</ParamField>
<ParamField body="funnel.settings" type="object">Configurações customizadas (opcional)</ParamField>
<ParamField body="funnel.global_custom_attributes" type="array">Atributos válidos pra todos os itens deste funil</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/funnels \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "funnel": {
      "name": "Vendas SMB",
      "description": "Pipeline comercial pra pequenas e médias empresas",
      "active": true,
      "is_default": false,
      "stages": {
        "lead": {
          "id": "lead",
          "name": "Lead",
          "color": "#94a3b8",
          "position": 1,
          "description": "Primeiro contato"
        },
        "qualified": {
          "id": "qualified",
          "name": "Qualificado",
          "color": "#3b82f6",
          "position": 2,
          "description": "Passou pelo SDR"
        },
        "proposal": {
          "id": "proposal",
          "name": "Proposta",
          "color": "#22c55e",
          "position": 3,
          "description": "Oferta enviada"
        },
        "won": {
          "id": "won",
          "name": "Ganho",
          "color": "#16a34a",
          "position": 4
        },
        "lost": {
          "id": "lost",
          "name": "Perdido",
          "color": "#dc2626",
          "position": 5
        }
      }
    }
  }'
```

### Resposta `201`


## Atualizar funil

```http
PATCH /funnels/{id}
PUT /funnels/{id}
```

<ParamField path="id" type="integer" required>ID do funil</ParamField>

Todos os campos do corpo são opcionais. Você manda apenas o que quer atualizar.

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/funnels/10 \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "funnel": {
      "name": "Vendas SMB 2026",
      "is_default": true
    }
  }'
```

### Resposta `200`

```json
{
  "id": 10,
  "name": "Vendas SMB 2026",
  "is_default": true,
  "updated_at": "2026-05-18T15:30:00Z"
}
```

## Remover funil

```http
DELETE /funnels/{id}
```

<ParamField path="id" type="integer" required>ID do funil</ParamField>

<Warning>
  `DELETE` é **irreversível** e remove em cascata todos os cards, notas,
  checklists e anexos vinculados. Considere arquivar via `active: false` em vez de deletar.
</Warning>

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/funnels/10 \
  -H "api_access_token: $TOKEN"
```

### Resposta `204`

Sem corpo. Apenas o status HTTP `204 No Content`.

## Reordenar etapas

```http
PATCH /funnels/{id}/reorder
```

<ParamField path="id" type="integer" required>ID do funil</ParamField>
<ParamField body="stages" type="array" required>Array de IDs de stages na nova ordem desejada</ParamField>

A ordem do array é a nova ordem visual. Todos os IDs precisam existir no funil.

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/funnels/10/reorder \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "stages": ["proposal", "lead", "qualified", "won", "lost"]
  }'
```

### Resposta `200`

```json
{
  "funnel_id": 10,
  "stages": [
    { "id": "proposal", "position": 1 },
    { "id": "lead", "position": 2 },
    { "id": "qualified", "position": 3 },
    { "id": "won", "position": 4 },
    { "id": "lost", "position": 5 }
  ]
}
```

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem, leitura ou atualização bem-sucedida |
| `201 Created` | Funil criado |
| `204 No Content` | Funil removido |
| `400 Bad Request` | Payload inválido (stages mal formado, IDs ausentes) |
| `401 Unauthorized` | Token ausente ou inválido |
| `403 Forbidden` | Sem permissão na conta |
| `404 Not Found` | Funil não existe |
| `422 Unprocessable Entity` | Validação falhou (ex: stage IDs duplicados, name vazio) |
