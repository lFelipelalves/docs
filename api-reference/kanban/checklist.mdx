---
title: "Checklist do card"
description: "Tarefas (to-do) dentro de cada card do Kanban, com agente, prazo e progresso."
---

Cada card pode ter uma lista de tarefas. Útil pra acompanhar etapas de qualificação,
documentos pendentes, follow-ups ou onboarding.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Criar tarefa

```http
POST /kanban_items/{id}/create_checklist_item
```

<ParamField path="id" type="integer" required>ID do card</ParamField>
<ParamField body="text" type="string" required>Descrição da tarefa</ParamField>
<ParamField body="due_date" type="date">Prazo no formato ISO (`YYYY-MM-DD`)</ParamField>
<ParamField body="priority" type="enum">`low`, `medium`, `high`, `urgent`</ParamField>
<ParamField body="agent_id" type="integer">Agente responsável pela tarefa</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_items/101/create_checklist_item \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Enviar contrato pra assinatura",
    "due_date": "2026-05-25",
    "priority": "high",
    "agent_id": 7
  }'
```

### Resposta `201`

```json
{
  "id": 55,
  "kanban_item_id": 101,
  "text": "Enviar contrato pra assinatura",
  "due_date": "2026-05-25",
  "priority": "high",
  "agent_id": 7,
  "completed": false,
  "created_at": "2026-05-18T17:30:00Z"
}
```

## Marcar / desmarcar conclusão

```http
POST /kanban_items/{id}/toggle_checklist_item
```

Toggle — se está aberto, fecha; se está fechado, reabre.

<ParamField path="id" type="integer" required>ID do card</ParamField>
<ParamField body="checklist_item_id" type="integer" required>ID da tarefa</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_items/101/toggle_checklist_item \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "checklist_item_id": 55 }'
```

### Resposta `200`

```json
{
  "id": 55,
  "completed": true,
  "completed_at": "2026-05-18T17:50:00Z",
  "completed_by_agent_id": 7
}
```

## Duplicar checklist pra outro card

```http
POST /kanban_items/{id}/duplicate_checklist
```

Útil quando você tem um card-template com processo padrão (ex: "Onboarding cliente novo")
e quer aplicar em outros cards.

<ParamField path="id" type="integer" required>Card de origem (de onde copiar)</ParamField>
<ParamField body="target_item_id" type="integer" required>Card destino (onde colar)</ParamField>
<ParamField body="merge" type="boolean" default="false">`false` substitui checklist do destino, `true` adiciona ao existente</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_items/101/duplicate_checklist \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "target_item_id": 202,
    "merge": false
  }'
```

### Resposta `200`

```json
{
  "source_item_id": 101,
  "target_item_id": 202,
  "items_copied": 5,
  "mode": "replace"
}
```

## Buscar texto na checklist

```http
GET /kanban_items/{id}/search_checklist
```

<ParamField path="id" type="integer" required>ID do card</ParamField>
<ParamField query="query" type="string" required>Texto a buscar</ParamField>

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_items/101/search_checklist?query=contrato" \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "query": "contrato",
  "matches": [
    { "id": 55, "text": "Enviar contrato pra assinatura", "completed": false }
  ],
  "count": 1
}
```

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem / atualização / toggle / assign |
| `201 Created` | Tarefa criada |
| `204 No Content` | Tarefa excluída |
| `400 Bad Request` | Payload inválido |
| `404 Not Found` | Card ou checklist_item_id não existe |
| `422 Unprocessable Entity` | `due_date` inválida, agente não pertence à conta |
