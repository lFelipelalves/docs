Automações de Kanban executam ações quando cards entram/saem de etapas
ou mudam de status. Tudo configurável via API.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Obter configuração atual

```http
GET /kanban_config
```

### Requisição

```bash cURL
curl https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_config \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "id": 1,
  "enabled": true,
  "webhook_url": "https://seu-n8n.com/webhook/kanban-events",
  "webhook_events": ["kanban.item.created", "kanban.item.stage_changed"],
  "config": {
    "title": "Kanban LF",
    "default_view": "kanban",
    "auto_assignment": false,
    "notifications_enabled": true,
    "dragbar_enabled": true,
    "list_view_enabled": true,
    "agenda_view_enabled": true
  }
}
```

---

## Criar configuração

```http
POST /kanban_config
```

### Corpo da requisição

<ParamField body="kanban_config.enabled" type="boolean" required>Liga/desliga o módulo Kanban</ParamField>
<ParamField body="kanban_config.webhook_url" type="string">URL externa que recebe eventos</ParamField>
<ParamField body="kanban_config.webhook_secret" type="string">Usado pra HMAC-SHA256 (validação de assinatura)</ParamField>
<ParamField body="kanban_config.webhook_events" type="array">Eventos que disparam o webhook</ParamField>
<ParamField body="kanban_config.config" type="object">Configurações de UI/comportamento (ver abaixo)</ParamField>

#### `config`

<ParamField body="config.title" type="string">Título exibido no menu Kanban</ParamField>
<ParamField body="config.default_view" type="enum">`kanban` ou `list`</ParamField>
<ParamField body="config.auto_assignment" type="boolean">Atribui automático ao criar card</ParamField>
<ParamField body="config.notifications_enabled" type="boolean">Notifica agentes em mudanças</ParamField>
<ParamField body="config.dragbar_enabled" type="boolean">Permite drag-and-drop visual</ParamField>
<ParamField body="config.list_view_enabled" type="boolean">Permite visão em lista</ParamField>
<ParamField body="config.agenda_view_enabled" type="boolean">Permite visão de agenda</ParamField>

#### Eventos disponíveis em `webhook_events`

- `kanban.item.created`
- `kanban.item.updated`
- `kanban.item.stage_changed`
- `kanban.item.status_changed`
- `kanban.checklist.completed`
- `kanban.note.created`

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_config \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "kanban_config": {
      "enabled": true,
      "webhook_url": "https://seu-n8n.com/webhook/kanban",
      "webhook_secret": "minha-secret-segura",
      "webhook_events": ["kanban.item.created", "kanban.item.stage_changed"],
      "config": {
        "title": "Kanban LF",
        "default_view": "kanban",
        "auto_assignment": false,
        "notifications_enabled": true,
        "dragbar_enabled": true,
        "list_view_enabled": true,
        "agenda_view_enabled": false
      }
    }
  }'
```

### Resposta `201`

```json
{
  "id": 1,
  "enabled": true,
  "webhook_url": "https://seu-n8n.com/webhook/kanban",
  "config": { "title": "Kanban LF", "default_view": "kanban" },
  "created_at": "2026-05-18T20:00:00Z"
}
```

## Atualizar configuração

```http
PUT /kanban_config
PATCH /kanban_config
```

Envie apenas os campos que quer mudar.

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_config \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "kanban_config": {
      "webhook_events": ["kanban.item.created", "kanban.item.stage_changed", "kanban.item.status_changed"]
    }
  }'
```

### Resposta `200`

```json
{
  "id": 1,
  "webhook_events": ["kanban.item.created", "kanban.item.stage_changed", "kanban.item.status_changed"],
  "updated_at": "2026-05-18T20:15:00Z"
}
```

---

## Excluir configuração

```http
DELETE /kanban_config
```

Volta o módulo Kanban pro padrão da plataforma (desativa webhook customizado).

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_config \
  -H "api_access_token: $TOKEN"
```

### Resposta `204`

Sem corpo.

## Testar webhook

```http
POST /kanban_config/test_webhook
```

Dispara um POST simulado pra `webhook_url` cadastrada — útil pra validar
que a URL responde corretamente.

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban_config/test_webhook \
  -H "api_access_token: $TOKEN"
```

### Resposta `200` — webhook respondeu OK

```json
{
  "ok": true,
  "webhook_url": "https://seu-n8n.com/webhook/kanban",
  "status_code": 200,
  "response_time_ms": 124,
  "test_payload": {
    "event": "test.webhook",
    "timestamp": "2026-05-18T20:30:00Z"
  }
}
```

### Resposta `200` — webhook falhou

```json
{
  "ok": false,
  "webhook_url": "https://seu-n8n.com/webhook/kanban",
  "status_code": 503,
  "error": "Service Unavailable",
  "response_time_ms": 5042
}
```

# Automações por etapa

Existem 3 variantes de endpoint. **Use sempre a primeira (escopo de conta)** —
as outras são mantidas por compatibilidade.

---

## Criar automação

```http
POST /kanban/automations
```

<ParamField body="kanban_automation.name" type="string" required>Nome legível</ParamField>
<ParamField body="kanban_automation.funnel_id" type="integer" required>Funil onde aplicar</ParamField>
<ParamField body="kanban_automation.stage_id" type="string" required>Etapa que dispara</ParamField>
<ParamField body="kanban_automation.trigger" type="enum" required>`stage_entered`, `stage_exited`, `item_created`, `item_status_changed`</ParamField>
<ParamField body="kanban_automation.conditions" type="array">Filtros opcionais (formato igual ao de regras de automação gerais)</ParamField>
<ParamField body="kanban_automation.actions" type="array" required>Ações a executar quando dispara</ParamField>
<ParamField body="kanban_automation.active" type="boolean" default="true">Liga/desliga</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/kanban/automations \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "kanban_automation": {
      "name": "Avisar comercial sobre proposta de alto valor",
      "funnel_id": 10,
      "stage_id": "proposal",
      "trigger": "stage_entered",
      "conditions": [
        { "attribute_key": "value", "filter_operator": "greater_than", "values": [10000] }
      ],
      "actions": [
        { "action_name": "send_message", "action_params": ["⚠️ Proposta acima de R$10k criada — avisar gerente!"] },
        { "action_name": "assign_team", "action_params": [3] }
      ],
      "active": true
    }
  }'
```

### Resposta `201`

```json
{
  "id": 5,
  "name": "Avisar comercial sobre proposta de alto valor",
  "active": true,
  "created_at": "2026-05-18T20:45:00Z"
}
```

## Endpoints globais legados

Mantidos por compatibilidade. **Não use em integrações novas.**

| Método | Endpoint |
| --- | --- |
| `GET` | `/api/v1/kanban_automations` |
| `GET` | `/api/v1/kanban_automations/:id` |
| `POST` | `/api/v1/kanban_automations` |
| `PATCH` `PUT` | `/api/v1/kanban_automations/:id` |
| `DELETE` | `/api/v1/kanban_automations/:id` |

Mesma estrutura de payload dos endpoints com escopo de conta. A única diferença
é a ausência do prefixo `/accounts/{account_id}` na rota.

---

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Listagem / leitura / atualização / teste de webhook |
| `201 Created` | Configuração ou automação criada |
| `204 No Content` | Removido |
| `400 Bad Request` | Payload inválido (trigger desconhecido, ação mal formada) |
| `403 Forbidden` | Sem permissão (apenas administrador) |
| `404 Not Found` | Automação ou configuração não existe |
| `422 Unprocessable Entity` | `stage_id` não existe no funil, action_name desconhecida |
