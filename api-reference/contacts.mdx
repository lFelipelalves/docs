---
title: "Contatos"
description: "CRUD de contatos, busca, importação em massa, notas, mesclagem e conversas vinculadas."
---

Contatos são as pessoas que conversam com sua equipe. Cada contato pode ter
múltiplas conversas (uma por canal/inbox) e atributos personalizados ilimitados.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Listar contatos

```http
GET /contacts
```

<ParamField query="q" type="string">Busca textual em nome, e-mail, telefone, identifier</ParamField>
<ParamField query="page" type="integer" default="1">Paginação (15 itens por página)</ParamField>
<ParamField query="sort" type="enum">`name`, `phone_number`, `email`, `created_at` (use `-name` pra desc)</ParamField>
<ParamField query="include_contact_inboxes" type="boolean">`true` retorna também as inboxes do contato</ParamField>

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v1/accounts/1/contacts?q=carlos&page=1" \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "meta": { "count": 1, "current_page": 1 },
  "payload": [
    {
      "id": 542,
      "name": "Carlos Antônio",
      "email": "carlos@empresa.com.br",
      "phone_number": "+5521999544338",
      "identifier": "crm-id-1234",
      "thumbnail": "https://cdn.lfautomatiza.com/...",
      "blocked": false,
      "additional_attributes": {
        "city": "Rio de Janeiro",
        "country": "Brazil",
        "company_name": "ACME Corp"
      },
      "custom_attributes": {
        "categoria": "vip"
      },
      "last_activity_at": 1715972400
    }
  ]
}
```

## Criar contato

```http
POST /contacts
```

<ParamField body="inbox_id" type="integer" required>Onde o contato será criado</ParamField>
<ParamField body="name" type="string">Nome do contato</ParamField>
<ParamField body="email" type="string">E-mail (único por conta)</ParamField>
<ParamField body="phone_number" type="string">E.164 (`+5521999544338`)</ParamField>
<ParamField body="identifier" type="string">ID externo (ex: ID do CRM)</ParamField>
<ParamField body="avatar_url" type="string">URL pública de imagem</ParamField>
<ParamField body="additional_attributes" type="object">Metadata sistêmica</ParamField>
<ParamField body="custom_attributes" type="object">Atributos personalizados definidos na conta</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/contacts \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "inbox_id": 1,
    "name": "Maria Silva",
    "email": "maria@empresa.com",
    "phone_number": "+5511999887766",
    "identifier": "crm-9876",
    "custom_attributes": {
      "fonte": "google-ads",
      "categoria": "lead-quente"
    }
  }'
```

### Resposta `200`

```json
{
  "id": 543,
  "name": "Maria Silva",
  "email": "maria@empresa.com",
  "phone_number": "+5511999887766",
  "identifier": "crm-9876",
  "contact_inboxes": [
    { "source_id": "+5511999887766", "inbox": { "id": 1, "name": "WhatsApp Principal" } }
  ]
}
```


```bash cURL
  -d '{
```

## Excluir contato

```http
DELETE /contacts/{id}
```

<ParamField path="id" type="integer" required>ID do contato</ParamField>

<Warning>
  Exclusão é irreversível e remove **todas as conversas e mensagens** vinculadas.
  Considere usar bloqueio em vez de exclusão.
</Warning>

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/contacts/542 \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{ "success": true }
```

## Filtro avançado

```http

```

<ParamField body="payload" type="array" required>Array de condições (estrutura igual ao filtro de conversas)</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/contacts/filter \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "payload": [
      {
        "attribute_key": "country",
        "filter_operator": "equal_to",
        "values": ["Brazil"]
      },
      {
        "attribute_key": "custom_attributes.categoria",
        "filter_operator": "equal_to",
        "values": ["vip"],
        "query_operator": "and"
      }
    ]
  }'
```

### Resposta `200`

```json
{
  "meta": { "count": 12 },
  "payload": [
    { "id": 542, "name": "Carlos Antônio", "custom_attributes": { "categoria": "vip" } }
  ]
}
```

## Importar contatos via CSV

```http
POST /contacts/import
```

Multipart. Colunas suportadas: `name`, `email`, `phone_number`, `identifier`,
`additional_attributes_*`, `custom_attributes_*`.

<ParamField body="import_file" type="file" required>Arquivo CSV</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/contacts/import \
  -H "api_access_token: $TOKEN" \
  -F "import_file=@contatos.csv"
```

### Resposta `200`

```json
{
  "import_id": "imp_xyz789",
  "status": "processing",
  "total_rows": 1247,
  "estimated_seconds": 60
}
```

<Tip>
  Tamanho recomendado: até 50.000 linhas por arquivo. Acima disso, divida.
</Tip>

```bash cURL
  -H "api_access_token: $TOKEN"
```

```bash cURL
  -H "api_access_token: $TOKEN"
```

```json
{ "id": 542, "blocked": false }
```

## Mesclar contatos (merge)

```http
POST /actions/contact_merge
```

Quando o mesmo cliente aparece em 2 cadastros separados (ex: WhatsApp + e-mail).

<ParamField body="base_contact_id" type="integer" required>Contato que será mantido</ParamField>
<ParamField body="mergee_contact_id" type="integer" required>Contato que será absorvido</ParamField>

Todas as conversas, notas e atributos do `mergee_contact_id` migram pro `base_contact_id`.
O contato mergee é removido após a operação.

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/actions/contact_merge \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "base_contact_id": 542,
    "mergee_contact_id": 543
  }'
```

### Resposta `200`

```json
{
  "id": 542,
  "name": "Carlos Antônio",
  "merged_count": 1,
  "conversations_migrated": 3,
  "notes_migrated": 2
}
```

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Sucesso em GET/POST/PATCH |
| `201 Created` | Contato criado |
| `204 No Content` | Nota excluída |
| `400 Bad Request` | Payload inválido |
| `403 Forbidden` | Sem permissão |
| `404 Not Found` | Contato não existe |
| `422 Unprocessable Entity` | E-mail ou telefone já existe na conta |
