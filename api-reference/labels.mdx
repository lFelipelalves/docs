---
title: "Labels (etiquetas)"
description: "CRUD de labels usadas em conversas e contatos."
---

Labels (também chamadas de tags ou etiquetas) são marcadores reutilizáveis para
organizar conversas e contatos. Cada label tem cor própria e pode aparecer em
filtros, automações e relatórios.

**Base:** `https://chat.lfautomatiza.com/api/v1/accounts/{account_id}`

---

## Listar labels

```http
GET /labels
```

### Requisição

```bash cURL
curl https://chat.lfautomatiza.com/api/v1/accounts/1/labels \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "payload": [
    {
      "id": 1,
      "title": "financeiro",
      "description": "Tickets relacionados a boleto, NF, pagamento",
      "color": "#9333ea",
      "show_on_sidebar": true
    },
    {
      "id": 2,
      "title": "vip",
      "description": "Clientes prioritários",
      "color": "#f59e0b",
      "show_on_sidebar": true
    }
  ]
}
```

## Criar label

```http
POST /labels
```

<ParamField body="title" type="string" required>Nome (sem espaço, lowercase recomendado)</ParamField>
<ParamField body="description" type="string">Tooltip / contexto</ParamField>
<ParamField body="color" type="string">Hex (`#00B86B`)</ParamField>
<ParamField body="show_on_sidebar" type="boolean">Aparece como atalho no menu lateral</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/labels \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "urgente",
    "description": "Casos que precisam de SLA <30min",
    "color": "#dc2626",
    "show_on_sidebar": true
  }'
```

### Resposta `201`

```json
{
  "id": 3,
  "title": "urgente",
  "description": "Casos que precisam de SLA <30min",
  "color": "#dc2626",
  "show_on_sidebar": true,
  "created_at": "2026-05-18T21:45:00Z"
}
```

## Atualizar label

```http
PATCH /labels/{id}
PUT /labels/{id}
```

<ParamField path="id" type="integer" required>ID da label</ParamField>

### Requisição

```bash cURL
curl -X PATCH https://chat.lfautomatiza.com/api/v1/accounts/1/labels/3 \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Casos críticos — SLA primeira resposta <15min",
    "color": "#b91c1c"
  }'
```

### Resposta `200`

```json
{
  "id": 3,
  "title": "urgente",
  "description": "Casos críticos — SLA primeira resposta <15min",
  "color": "#b91c1c"
}
```

## Excluir label

```http
DELETE /labels/{id}
```

<ParamField path="id" type="integer" required>ID da label</ParamField>

<Note>
  Excluir uma label remove ela de **todas** as conversas e contatos marcados.
  Sem "undo" — considere arquivar (renomear pra `_zarchive_xxx`) em vez de excluir.
</Note>

### Requisição

```bash cURL
curl -X DELETE https://chat.lfautomatiza.com/api/v1/accounts/1/labels/3 \
  -H "api_access_token: $TOKEN"
```

### Resposta `204`

Sem corpo.

## Aplicar label numa conversa

```http
POST /conversations/{id}/labels
```

O array enviado **substitui** todas as labels atuais da conversa.

<ParamField path="id" type="integer" required>ID da conversa</ParamField>
<ParamField body="labels" type="array" required>Lista completa de labels desejadas</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/conversations/1287/labels \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "labels": ["financeiro", "vip"] }'
```

### Resposta `200`

```json
{ "payload": ["financeiro", "vip"] }
```

---

## Aplicar label num contato

```http
POST /contacts/{id}/labels
```

Mesma estrutura — array substitui o conjunto atual.

<ParamField path="id" type="integer" required>ID do contato</ParamField>
<ParamField body="labels" type="array" required>Lista completa de labels desejadas</ParamField>

### Requisição

```bash cURL
curl -X POST https://chat.lfautomatiza.com/api/v1/accounts/1/contacts/542/labels \
  -H "api_access_token: $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "labels": ["vip"] }'
```

### Resposta `200`

```json
{ "payload": ["vip"] }
```

## Filtrar conversas por label

```http
GET /conversations?labels[]={label1}&labels[]={label2}
```

Múltiplas labels = combinação **AND**.

### Requisição

```bash cURL
curl "https://chat.lfautomatiza.com/api/v1/accounts/1/conversations?labels[]=vip&labels[]=urgente" \
  -H "api_access_token: $TOKEN"
```

### Resposta `200`

```json
{
  "data": {
    "meta": { "all_count": 4 },
    "payload": [
      { "id": 1287, "labels": ["financeiro", "vip", "urgente"] }
    ]
  }
}
```

---

## Códigos de resposta

| Código | Quando acontece |
| --- | --- |
| `200 OK` | Sucesso em GET / PATCH / aplicação em conversa-contato |
| `201 Created` | Label criada |
| `204 No Content` | Excluída |
| `422 Unprocessable Entity` | Título já existe na conta ou cor inválida |
